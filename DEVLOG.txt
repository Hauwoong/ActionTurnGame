- 카드 효과 처리 로직 설계 및 테스트

### 문제
- CardData 추상 클래스와 virtual / abstract 혼용으로 컴파일 오류 발생
- BattleContext 전달 구조 설계 미흡으로 UseCard 호출 오류 발생

### 해결
- CardData 구조를 추상 기반 공통 처리 + 데이터 중심 구조로 재설계
- 카드 사용 시 BattleContext를 통해 전투 정보 전달하도록 구조 수정
- 카드 타입 분류(enum) 제거 → 주사위 데이터 중심 구조로 통합
- 기본 카드와 특수 카드 상속 구조 방향 정립
- 카드 역할을 주사위 조합으로 표현하는 방식 확정

### 배운 점
- 카드 시스템의 핵심은 “카드 타입”이 아니라 “행동 데이터”이다
- ScriptableObject + 데이터 중심 구조는 확장성이 매우 높다
- abstract / virtual은 역할이 명확할 때만 사용하는 것이 좋다
- BattleContext 같은 컨텍스트 객체는 시스템 결합도를 낮춰준다
- 중복 구조를 의심하고 제거하는 설계가 장기 유지보수에 중요하다
