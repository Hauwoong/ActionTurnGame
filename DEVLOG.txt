## 2026-01-26

### 오늘 한 것
-TurnManager에 TurnState(enum) 기반 턴 시스템 구조 완성
-턴 변경 이벤트(OnTurnChanged) 구현 및 UI 연동
-TurnUI / HPUI 스크립트 분리 및 구조화
-플레이어 HP / Energy 캡슐화 적용
-UI 자동 갱신 시스템 설계 및 테스트
-RectTransform Anchor / Pivot 기반 UI 배치 학습

###문제
-TurnManager 이벤트 인식 오류(CS1061) 발생
-UI 위치 이동 시 좌표 변경이 정상적으로 적용되지 않음
-에너지 / 체력 UI 갱신 구조 설계에 대한 혼란
-UI 통합 관리 vs 개별 관리 구조 고민 발생

###해결
-TurnManager에 이벤트(Action<TurnState>) 직접 선언 및 연결
-RectTransform Anchor Preset(Shift + Alt) 방식으로 UI 재배치
-이벤트 기반 UI 갱신 구조로 전환
-UI 기능별 스크립트 분리로 책임 명확화
-Awake() 초기화 → UI 최초 갱신 안정화

###배운 점
-이벤트 패턴은 Update() 의존도를 줄이고 구조를 깔끔하게 만든다
-캡슐화를 적용하면 외부 접근 제어와 유지보수가 쉬워진다
-UI는 좌표보다 Anchor 구조가 더 중요하다
-기능별 UI 분리는 확장성을 크게 높인다
-구조 설계를 먼저 하면 디버깅 시간이 줄어든다
